03. React-III

● React-III

◎ 클래스형 컴포넌트
* static, 생명주기(lifeCycle) 관련 기능을 사용할 수 있다.
* this를 사용해서 state / props / refs / 컴포넌트 메소드 / 생명주기 메소드를 참조할 수 있다.
* 이 때 bind(this)를 사용해 클래스 내부의 요소를 명시하지 않으면 최상단의 window를 this로 인식하게
  되므로 주의가 필요하다. (화살표 함수로 함수를 선언하는 경우에는 bind( )를 사용하지 않아도 된다.)
* 함수형에 비해 상대적으로 메모리를 많이 사용한다.
* 컴포넌트를 사용하기 위해 react의 Component를 반드시 import 해줘야 한다.

◎ 함수형 컴포넌트
* state, lifeCycle 관련 기능은 사용할 수 없다.
* 리액트 훅(Hook)이 도입되며 위의 단점은 상쇄되었다. 
  (useState, useEffect 등을 사용해서 구현 가능해짐)
* 클래스형에 비해 상대적으로 적은 메모리를 사용한다.
* 컴포넌트 선언이 편하고 클래스형에 비해 심플한 구조를 가지고 있다.
  현재 리액트의 공식 문서는 함수형 컴포넌트 선언을 권장한다.
