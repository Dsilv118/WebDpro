16. Collection

● Collection
컬렉션이란, 우리말로 쉽게 말해서 자료구조입니다. 다수의 데이터, 즉 데이터그룹을 의미합니다.
데이터 군을 저장하기 위해서 우리는 배열을 사용해왔는데요, 배열과 비슷한 구조에다가 
다수의 데이터를 쉽게 처리할 수 있는 방법을 제공하는 API입니다.

◎ 자바에서 제공되는 Collection 자료구조들 
 
                                                       Collection

                              List                         Set                         Map
                      (순서, 중복 허용)           (순서x, 중복x)               (key, value)

                           ArrayList                  HashSet                    HashMap
                            Vector                    TreeSet                    Properties
                          LinkedList

※ List 계열 Collection 클래스 
자료구조 중 가장 많이 사용하고 쉽게 사용할 수 있는 자료구조형이 List계열입니다. List는 
배열과 비슷하지만, 배열의 단점을 보완하였습니다.

※ Map 계열 Collection 클래스
Map계열의 자료구조는 인덱스 대신 키 값으로 데이터를 엑세스합니다.
List계열과 달리 인덱스가 없고, 키와 값만 있습니다. 그리고 키는 유니크 해야 합니다. 우리가 값을
관리하고자 한다면 키를 이용해서 값을 관리할 수 있습니다.

※ Set 계열 Collection 클래스
Set계열 자료구조에서는 데이터의 순서는 없습니다(인덱스 없다).
하지만 중복된 데이터는 허락하지 않습니다. 
중복된 데이터의 의미는 hashcode() 값이 같거나 equal() 메소드의 결과값에 의해 해석합니다.

◎ iterator(반복자) 자료구조
JAVA에는 많은 자료구조형이 있다고 하였습니다. 많은 자료구조형이 존재하는 것은 결국은 데이터를
잘 관리하기 위해서입니다. 잘 관리한다는 것은 저장하고, 정렬하고, 검색하는 것입니다. 
JAVA에서는 Iterator라는 인터페이스가 있습니다. Iterator라는 의미는 '반복자'라는 의미로 데이터를 
반복적으로 검색하는데 아주 유용한 인터페이스입니다.
참고로 모든 자료구조형은 iterator() 메소드를 지원 하고 있습니다.
